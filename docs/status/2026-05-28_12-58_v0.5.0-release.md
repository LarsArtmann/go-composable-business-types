## D) TOTALLY FUCKED UP (Now Fixed)

### 1. Sub-Module go.mod Files Had No Dependency Declarations

**Status:** FIXED in session 1 (commit `13e59fc` on `modularize/split-modules`)

The initial modularization created sub-module go.mod files without declaring dependencies on the root module or sibling modules. Only worked through `go.work` coincidence.

### 2. `replace` Scaffolding Prevented Clean Release

**Status:** FIXED in this session

Sub-modules had `replace` directives pointing to local paths. These are ignored by external consumers, so a release would have failed for anyone importing the sub-modules.

### 3. `GOWORK = "off"` Broke Nix Builds

**Status:** FIXED in this session

The Nix devShell set `GOWORK = "off"`, which broke workspace-based builds in the Nix environment.

### 4. Nix Checks Only Tested Root Module

**Status:** FIXED in this session

The Nix `build`, `test`, and `lint` checks only ran against the root module, missing sub-module regressions.
